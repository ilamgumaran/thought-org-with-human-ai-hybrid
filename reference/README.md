# Reference Library

Structured knowledge extracted from external sources — videos, papers, talks, frameworks — used as building blocks for creating HIO transition plans tailored to specific org structures and roles.

## How This Directory Works

Each reference is stored as a standalone markdown file with:
- **Source metadata** (title, author, URL, date, type)
- **Key concepts** extracted and structured for reuse
- **Role mapping** — how the concepts map to engineering org roles
- **HIO integration notes** — how this connects to the HIO framework

## Usage

These references are consumed by transition plans in `examples/` to:
1. Define what skills/capabilities an org needs to develop
2. Map existing roles to new cognitive functions
3. Identify gaps between current and target state
4. Build role-specific learning paths
